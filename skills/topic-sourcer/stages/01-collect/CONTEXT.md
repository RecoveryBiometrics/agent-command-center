# Stage 1: Collect Topic Candidates

## Inputs
- Business ID (e.g., "globalhighlevel")
- Target language code (e.g., "es", "en-IN", "ar")
- Published records for this language (to skip already-covered topics)

## Process

### Tier 1: GHL Documentation
1. Read `articles-cache.json` from the pipeline data directory
   - Path: `{org.path}/ghl-podcast-pipeline/data/articles-cache.json`
   - Contains: ~2,961 GHL help article records with id, title, url, body, category
2. Filter out articles already covered in the target language
   - Check `{language}-published.json` for matching `articleId` values
3. Output: list of uncovered articles with full body text

### Tier 2: GSC Gap Queries
1. Read `gsc-stats.json` looking for the `queries_{language}` key
   - Example: `queries_es`, `queries_in`, `queries_ar`
   - These are generated by `analytics.py` with country-filtered GSC data
2. Find queries with:
   - Impressions >= 20
   - No matching page in the site posts for that language
3. If no language-specific GSC data exists yet, skip this tier
4. Output: list of gap queries with impressions and position data

### Tier 3: Market-Specific Topics
1. Read `references/language-config.md` for the target market's:
   - Key industries
   - Local competitors
   - Common business types
2. Read `references/tier3-markets.md` for topic generation patterns
3. Check already-published topics to avoid duplicates
4. Generate 5-10 candidate topics using Claude with market context
5. Output: list of market-specific topics with industry tags

## Outputs
Pass to Stage 2:
- `tier1_candidates` — GHL help articles not yet covered in this language
- `tier2_candidates` — GSC gap queries for this language (may be empty)
- `tier3_candidates` — Market-specific topics generated from industry lists

## Tools needed
- File reads for articles-cache.json, gsc-stats.json, published records
- Claude API for Tier 3 topic generation (if needed)
