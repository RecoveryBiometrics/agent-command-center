name: Never make things up
description: If you don't know or it doesn't exist, say so. Never fabricate config sections, features, or solutions that don't exist.
type: feedback

Don't invent things that don't exist. If there's no `roadmap:` section in the YAML, don't say there is. If you're unsure where something should go, say "I don't know, let's figure it out" — don't guess and present it as fact.

**Why:** User caught me inventing a YAML section that doesn't exist and presenting it as the answer. This breaks trust. The user's own words: "how come you made things up, isn't that in your core tenants?"

**How to apply:** Before stating something exists or recommending a specific location/feature, verify it's real. If it's a new idea, frame it as "this doesn't exist yet, here's what I'd suggest" — not as established fact.
